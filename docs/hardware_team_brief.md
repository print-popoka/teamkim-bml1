# 하드웨어팀 — 초음파 센서 2개 살려주세요

소프트팀 진단 결과를 1페이지로 요약. 멀티미터 꺼내기 전 **눈으로 먼저 확인할 체크리스트**.

## 비유로 이해하기

HC-SR04 초음파 센서는 사실 **전화기랑 똑같아요.** 선이 4개:

| 선 | 역할 | 비유 |
|---|---|---|
| **VCC (5V)** | 전원 | 전화기 충전 |
| **GND** | 접지 | 전화기 케이스 (기준점) |
| **TRIG** | 출력 (Pi → 센서) | "전화 걸어!" 명령 |
| **ECHO** | 입력 (센서 → Pi) | "통화 연결됐어!" 응답 |

정상 동작:
1. Pi가 TRIG를 짧게 누름 ("거리 재줘")
2. 센서가 초음파를 쏨 (찌익)
3. 벽에 부딪혀 돌아옴
4. 센서가 ECHO를 짧게 HIGH로 올림 ("시간은 N초였어")
5. Pi가 시간 보고 거리 계산

## 우리 현재 상태

3개 센서 중 **앞쪽 1개만 정상, 나머지 2개 죽음.**

```
[OK]  FRONT  (앞 정면)   → 9.8cm 정상 출력
[FAIL] LEFT45 (좌측 45°)  → 전화기가 아예 안 켜짐
[FAIL] RIGHT45(우측 45°)  → 전화기가 신호 보내지도 않았는데 계속 통화중
```

## LEFT45 — "전화기가 안 켜져요"

**증상**: ECHO 응답이 아예 안 옴 (`echo never went HIGH`)
**의미**: 센서가 초음파를 못 쏘거나, 응답선이 끊김
**가장 흔한 원인**: **VCC(빨간 전선) 안 꽂힘 → 센서 전원 안 들어옴**

### 눈으로 1분 점검

1. LEFT45 센서의 4개 핀에 선 4개가 다 꽂혀있나? (꼭 4개)
2. **VCC선이 Pi의 5V 핀에 정말 연결됐나?** (가장 흔한 빠짐)
3. GND선이 Pi의 GND 핀에 연결됐나?
4. TRIG선이 **GPIO 25** (Pi 헤더 22번 핀)
5. ECHO선이 **GPIO 8** (Pi 헤더 24번 핀)

→ 4개 선 다 한 번씩 빼서 다시 꽂아보기. 80% 확률로 이걸로 해결됩니다.

## RIGHT45 — "전화기가 계속 통화중"

**증상**: ECHO 응답선이 idle 상태에서도 계속 HIGH (`echo idle HIGH`)
**의미**: 응답선이 안 떨어짐 — 보통 **접지(GND) 안 됨**
**가장 흔한 원인**: **GND선 빠짐 → 응답선이 공중에 떠서 5V로 끌려감**

### 눈으로 1분 점검

1. RIGHT45 센서의 4개 핀에 선 4개 다 꽂혀있나?
2. **GND선이 Pi의 GND 핀에 정말 연결됐나?** (가장 흔한 원인)
3. **ECHO선(보통 노란색)이 VCC선(빨간색)에 닿아있지 않나?** 살짝 옆 핀이라 자주 닿음
4. TRIG선이 **GPIO 7** (Pi 헤더 26번 핀)
5. ECHO선이 **GPIO 12** (Pi 헤더 32번 핀)

→ 마찬가지로 4개 선 다 한 번씩 빼서 다시 꽂기.

## FRONT 센서가 정상인 게 단서

FRONT(GPIO 23/24)는 동일한 전원·접지 구성에서 잘 동작합니다.
→ Pi 자체나 전체 배선 컨셉의 문제가 아니라 **LEFT45·RIGHT45의 개별 배선 문제**.
